restart....


	--restart flag in run use to set the restart policy for a container 

common restart policy:

	no 		 default (Do not restart the container automatically)
	always		always restart 	the container if it stops
	on failure	restart only if it container exits with non zero status
	unless-stopped  restart the unless you manually stop the conatiner

	docker run -d --name webserver --restart always nginx
	if the container crashes or Docker daemon restarts, it will start again.

	docker run -d --name app --restart on-failure:3 ubuntu sleep 10
	Will retry up to 3 times if the container fails.

	docker run -d --name db --restart unless-stopped mysql
	Keeps restarting unless you explicitly stop it.



--entrypoint <command>    Overrides the default entrypoint of the image.

	docker run --entrypoint /bin/bash -it ubuntu

	Normally, Ubuntu runs /bin/sh as the default shell.
	Here, we override it to use /bin/bash.


--workdir <path>       Sets the working directory inside the container.


	When you enter the container, your current directory will be /app.
	If /app doesn’t exist, Docker creates it.


 --privileged  Gives the container extended privileges (like full access to the host system).

	docker run -it --privileged ubuntu bash

	Useful for running Docker inside Docker or accessing hardware.
	 Use with caution because it bypasses many security restrictions.
